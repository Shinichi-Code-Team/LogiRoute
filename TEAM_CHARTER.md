## Member 2: Ghada Al-Saqqa

## Coding Standards

### Purpose
To maintain a clean, consistent, and maintainable codebase, all team members agree to follow the coding standards below.
These guidelines improve collaboration, readability, and simplify code reviews.

### Naming Conventions
- Use camelCase for variables and functions.
- Use PascalCase for classes.
- Use UPPER_SNAKE_CASE for constants.
- Use meaningful and descriptive names.

### Code Quality
- Keep each function focused on a single responsibility.
- Follow the DRY (Don't Repeat Yourself) principle.
- Prefer simple, readable, and efficient solutions.
- Remove unused code before committing.
- Handle errors whenever necessary.

### Formatting & Documentation
- Use consistent indentation and formatting.
- Write comments only when they improve understanding.
- Keep files and folders well organized.
- Write clear commit messages describing your changes.

### Git Workflow
- Create a separate branch for every feature or task.
- Never commit directly to the `main` branch.
- Pull the latest changes before starting new work.
- Open a Pull Request before merging into `main`.
- Resolve merge conflicts before requesting a review.

### Team Commitment
All team members are responsible for following these standards to ensure a clean, collaborative, and maintainable project.
